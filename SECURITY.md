# Security

## API Key Rules

Never commit API keys.

Forbidden:

- API keys in code
- API keys in README files
- API keys in `.env`
- API keys in `frontend/public`
- `NEXT_PUBLIC_FINNHUB_API_KEY`
- frontend requests directly to Finnhub, FRED, or other paid providers

Allowed:

- GitHub Secrets:
  - `FINNHUB_API_KEY`
  - `FRED_API_KEY`
- Local `.env` files that are ignored by git
- Backend/script-only provider calls during GitHub Actions

## Frontend Data Boundary

The dashboard reads only static JSON generated by the scripts:

- `stock-radar-dashboard.json`
- `top-candidates.json`
- `stock-forecast-records.json`
- `validation-scorecard.json`
- `stock-model-leaderboard.json`

If a value is visible in these files, assume it is public.

## Product Safety Boundary

This repository must not implement trading execution or present outputs as instructions.

Forbidden:

- automated trading
- order placement
- directional order wording as an instruction
- position sizing
- guaranteed profit, guaranteed rise, or guaranteed fall claims
- promotion of low-liquidity junk names
- fake data
- treating proxy data as real data
- presenting backtests as confirmed alpha before forward validation

Required page disclaimer:

```text
这是次日高弹性概率雷达，不是投资建议或交易指令。
```

## Data Freshness

If data is stale, missing, provider-failed, or fallback-only, the dashboard must display a warning. The system must not pretend stale data is a current-day forecast.

## Reporting Issues

If you find a leaked key or unsafe data boundary, rotate the affected key immediately and remove the leaked content from the repository history where practical.
